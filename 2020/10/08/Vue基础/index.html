Vue知识整理

发表于 2020-10-08 更新于 2020-10-11 分类于 学习 阅读次数：

vue基础

el：挂载点

  • el是用来设置Vue实例挂载（管理）的元素
  • Vue会管理el选项命中的元素及其内部的后代元素
  • 可以使用其他选择器，但是建议使用ID选择器，避免歧义
  • 可以使用其他双标签，不能使用HTML和BODY

data：数据对象

  • Vue中用到的数据定义在data中
  • data中可以写复杂的数据
  • 渲染复杂类型时遵循js语法即可

指令

v-text

  • v-text指令的作用：设置标签的内容（textContent）
  • 默认写法会替换标签内的全部内容，使用插值表达式{{}}可以替换指定内容
  • v-text指令的内容只会解析为文本
  • 无插值表达式的闪烁问题

v-cloak

  • v-cloak指令的作用：保持在元素上直到关联实例结束编译
  • 和 css规则如 [v-cloak] { display: none } 一起用时，解决插值表达式闪烁问题，隐藏未编译的 Mustache 标签直到实例准备完毕。

v-html

  • v-html指令的作用：设置元素的innerHTML
  • 内容中有html结构会被解析为标签

v-on

  • v-on指令的作用：为元素绑定事件

  • 指令可简写为 @（v-on:）

  • 绑定的方法定义在methods属性中

  • 方法内部使用this关键字可以访问定义在data中的数据

  • 绑定事件可以写成函数调用形式，可以传入自定义参数

  • 定义方法时需要定义形参来接收传入的实参

  • 事件后跟上修饰符可以对事件进行限制

    常见的事件修饰符：

    • .stop：阻止冒泡（从内到外）
    • .prevent：阻止默认事件（如标签等）
    • .capture：添加事件监听器时使用事件捕获模式（从外到内）
    • .self：只当事件在该元素本身出发时才触发回调
    • .once：事件只触发一次

    .self只会阻止自己身上的冒泡行为触发，并不能阻止冒泡本身

v-bind

  • v-bind指令的作用：为元素绑定属性
  • 简写为省略v-bind， :属性名
  • 动态增删有对象（{属性名：条件}）与三目运算（条件？属性名：属性名）两种方式

v-model

  • v-model指令的作用：双向绑定表单数据的值
  • v-model用于其他组件属于自定义指令

v-bind只能实现数据的单向绑定，从M自动绑定到V

v-model可以实现表单元素与model数据的双向绑定

v-show

  • v-show指令的作用：根据真假切换元素的显示状态
  • 原理为修改元素的display
  • 指令后的内容均解析为布尔值
  • true为显示，false为隐藏

v-if

  • v-if指令的作用：根据表达式的真假切换元素的显示状态
  • 原理为增删DOM元素来切换显示状态
  • true为添加进DOM树，false为从DOM树中移除

v-if切换性能消耗，v-show有较高的初始渲染消耗

v-for

  • v-for指令的作用：根据数据生成列表结构
  • 遍历数组(item,index) in 数组
  • 遍历对象时为(val,key,index)in 对象
  • 迭代数字（起始值为1） count in 数字

如无需要，通常不使用索引

在2.2.0+的版本中，当在组件中使用v-for时，key是必须的，用于指明唯一性，key属性只能使用number或string，且在使用时须使用v-bind属性绑定指定key的值

样式处理

v-bind绑定class

  • 直接传递数组

  • 直接传递对象

v-bind绑定style

  • 直接传递数组

  • 直接传递对象

Vue进阶

过滤器

Vue.js允许自定义过滤器，可被用于一些常见的文本格式化。

通常被用于mastache插值与v-bind表达式(后者从 2.1.0+ 开始支持)。

  • 全局过滤器

  • 组件过滤器

function的第一个参数为待过滤的文本值，可添加其他参数

当全局过滤器和局部过滤器重名时，会根据就近原则采用局部过滤器

自定义按键修饰符

键盘按键对应编码值

自定义全局指令

Vue 允许注册自定义指令。当需要对普通 DOM 元素进行底层操作，可以用到自定义指令。

Vue中所有的指令都以v-开头。

钩子函数

bind:function(el){}

  • 只调用一次，指令第一次绑定到元素时调用。在这里可以进行一次性的初始化设置，例如修改初始样式等。

inserted:function(el){}

  • 被绑定元素插入父节点时调用 (仅保证父节点存在，但不一定已被插入文档中)。

updated:function(el){}

  • 所在组件的 VNode 更新时调用，但是可能发生在其子 VNode 更新之前。指令的值可能发生了改变，也可能没有（可以通过比较更新前后的值来忽略不必要的模板更新）。

在元素刚绑定指令时，还未插入到DOM中，此时无法直接操作

bind通常进行样式相关的操作

inserted通常设置js行为相关的操作

钩子函数参数

指令钩子函数会被传入以下参数：

  • el：指令所绑定的元素，可以用来直接操作 DOM。
  • binding：一个对象，包含以下 property：
    • name：指令名，不包括 v- 前缀。
    • value：指令的绑定值，例如：v-my-directive="1 + 1" 中，绑定值为 2。
    • oldValue：指令绑定的前一个值，仅在 update 和 componentUpdated 钩子中可用。无论值是否改变都可用。
    • expression：字符串形式的指令表达式。例如 v-my-directive="1 + 1" 中，表达式为 "1 + 1"。
    • arg：传给指令的参数，可选。例如 v-my-directive:foo 中，参数为 "foo"。
    • modifiers：一个包含修饰符的对象。例如：v-my-directive.foo.bar 中，修饰符对象为 { foo: true, bar: true }。
  • vnode：Vue 编译生成的虚拟节点。
  • oldVnode：上一个虚拟节点，仅在 update 和 componentUpdated 钩子中可用。

除了 el 之外，其它参数都应该是只读的，切勿进行修改。如果需要在钩子之间共享数据，建议通过元素的dataset来进行

Vue实例的生命周期

不要在选项 property 或回调上使用箭头函数，比如

created: () => console.log(this.a)

vm.$watch('a', newValue => this.myMethod())。

因为箭头函数并没有 this，this 会作为变量一直向上级词法作用域查找，直至找到为止，经常导致

Uncaught TypeError: Cannot read property of undefined 或 Uncaught TypeError: this.myMethod is not a function 之类的错误

创建期间的生命周期函数

beforeCreate()

此函数执行于Vue实例刚开始创建时，此时刚初始化了一个空的Vue对象，此对象只包含一些默认的生命周期函数与默认的事件，其他内容尚未创建。

此时，data与methods中的数据还没有被初始化。

created()

此时，data与methods中的数据已经被初始化。

beforeMount()

此时，编译模板完成，把Vue代码中的指令进行执行，最终在内存中生成一个编译好的模板字符串，并渲染成内存中的DOM，但并未挂载到真正的页面中，处于挂载之前的状态。

此时页面中的元素还未被真正替换，只是模板字符串。

mounted()

创建实例期间的最后一个函数，此时页面已完成挂载，只要完成该函数，则表示Vue实例已完成初始化。

最早操作data与methods中的数据在created()中进行

最早通过某些组件操作页面上的DOM节点在mounted()中进行

运行期间的的生命周期函数

beforeUpdate()

此时数据被更新，data中的数据是更新后的，而界面尚未被更新，界面中的数据依旧为旧数据。

update()

此时VM根据data中最新的数据，重新渲染一份最新的内存DOM树，并挂载到真实的页面中，完成从data（Model层）到view（视图层）的更新

这两个生命周期函数会根据data数据的改变选择性的触发0次或者多次

销毁期间的生命周期函数

beforeDestroy()

此时，实例中所有的data，methods以及指令等仍处于可用状态，尚未真正执行销毁的过程。

destroy()

此时，实例所有的数据、方法、指令和过滤器等都已经不可用。

数据传输

vue-resource

引入vue-resource后，可以基于全局的Vue对象使用http，也可以基于某个Vue实例使用http。

在发送请求后，使用then方法来处理响应结果，then方法有两个参数，第一个参数是响应成功时的回调函数，第二个参数是响应失败时的回调函数。

vue-resource的请求API是按照REST风格设计的，它提供了7种请求API：

  • get(url, [options])
  • head(url, [options])
  • delete(url, [options])
  • jsonp(url, [options])
  • post(url, [body], [options])
  • put(url, [body], [options])
  • patch(url, [body], [options])

除了jsonp以外，另外6种的API名称是标准的HTTP方法。当服务端使用REST API时，客户端的编码风格和服务端的编码风格近乎一致，这可以减少前端和后端开发人员的沟通成本。

vue-resource依赖于Vue，要注意引用的先后顺序
