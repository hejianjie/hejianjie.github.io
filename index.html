ES6新特性整理

发表于 2020-10-11 更新于 2020-10-12 分类于 学习

官方文档

ES6文档，下面给出用到的较多的新特性。

变量声明

ES6推荐使用let声明局部变量，相比之前的var（Hoisting特性：无论声明在何处，都会被视为声明在函数的最顶部）

  • let与var

    • let具有块级作用域的概念，不存在Hoisting特性，即在最靠近的一个块中（花括号内）有效；

    • let不允许在同一作用域内重复声明同一个变量。

  • let与const

    • let用于声明局部变量，而const用于声明常量，两者都为块级作用域；

  • const 声明的变量都会被认为是常量，即它的值被设置完成后就不能再修改了；

  • 当使用常量 const 声明时，一般推荐使用大写变量；

  • const 在声明时必须被赋值，不能先声明，后初始化；

  • const声明的是一个对象时，对象所包含的值是可以被修改的。可以认为对象所指向的地址没有变就行：

    const和let都有暂时性死区：

    ES6规定，如果在区块中存在let和const声明的变量，则这个区块对这些声明的变量从一开始就形成一个封闭的作用域。不管在外部有没有声明这个变量。必须遵守“先声明，后使用”的原则，否则报错

    暂时性死区的本质就是，只要一进入当前作用域，所使用的变量就已存在，但是不可获取，只有等到声明变量的哪一行代码的出现，在可以获取和使用该变量。

箭头函数（Arrow Functions）

ES6中新增箭头操作符用于简化函数的写法，操作符左边为参数，右边为具体操作和返回值。

Vue知识整理

发表于 2020-10-08 更新于 2020-10-11 分类于 学习

vue基础

el：挂载点

  • el是用来设置Vue实例挂载（管理）的元素
  • Vue会管理el选项命中的元素及其内部的后代元素
  • 可以使用其他选择器，但是建议使用ID选择器，避免歧义
  • 可以使用其他双标签，不能使用HTML和BODY

data：数据对象

  • Vue中用到的数据定义在data中
  • data中可以写复杂的数据
  • 渲染复杂类型时遵循js语法即可

指令

v-text

  • v-text指令的作用：设置标签的内容（textContent）
  • 默认写法会替换标签内的全部内容，使用插值表达式{{}}可以替换指定内容
  • v-text指令的内容只会解析为文本
  • 无插值表达式的闪烁问题

v-cloak

  • v-cloak指令的作用：保持在元素上直到关联实例结束编译
  • 和 css规则如 [v-cloak] { display: none } 一起用时，解决插值表达式闪烁问题，隐藏未编译的 Mustache 标签直到实例准备完毕。

v-html

  • v-html指令的作用：设置元素的innerHTML
  • 内容中有html结构会被解析为标签

renren-fast使用笔记

发表于 2020-09-29 更新于 2020-10-07 分类于 模板&框架

前言

renren-fast是人人开源的一个快速开发平台的框架，实现了前后端分离，后台为Spring Boot，前端为Vue。整体分为三个部分：renren-fast（后台），renren-fast-vue（前端），renren-gennerator（代码生成器）。

目前个人使用这套框架搭建了一个管理系统，当然仅仅是搭建起来了，对于项目中集成的多种技术以及整体的开发逻辑还在艰难学习中。谨以此文记录一下项目搭建过程中遇到的一些问题。

同时附上一篇网上找到的踩坑记录，里面有不少有用的解决方案。

renren-fast

部署

后端部署就是常规的git操作，环境要求是JDK1.8 、 MySQL5.5+，然后等待依赖导入完成就可以更改具体的配置了。

项目的DB文件夹中是用于生成项目的SQL文件，自行创建数据库初始化数据，数据库编码为 UTF-8。创建完成后修改application-dev.yml配置文件 ，更新数据库账号和密码，然后运行 io.renren.RenrenApplication.java 的 main 方法，则可启动项目。总体来说，后端的部署基本不会出现什么问题，不能部署可以找一下项目环境相关的问题，例如jdk，maven，镜像之类的问题等。

注意事项

  1. 很多命令既可以用Windows的cmd来完成，也可以使用git bash来完成，但是部分命令会有一些问题，为避免不必要的问题，建议全部使用git bash来执行；

  2. hexo不同版本差别比较大，网上很多文章的配置信息都是基于2.x的，所以注意不要被误导；同时我个人使用的主题Next也有多个版本，截止此文撰写最新的版本为 8.0.0，往期的旧版本已停止维护，很多文章的对应配置信息也与新的版本有所出入，需要自行甄别；

  3. hexo有2种_config.yml文件，一个是根目录下的全局的_config.yml，一个是各个theme下的；

  4. 有的教程会让配置SSH密钥，SSH密钥可以为了防止其他人恶意部署文件到你的仓库，使用以后每次部署文件都需要登录Github账号，通过下述配置可比避免每次登录：
